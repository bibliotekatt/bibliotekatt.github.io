System Architecture & Policy Review

Academic Progression Review Periods

Proposed Stream Timings, Policy Alignment, and Operational Impact Analysis.

1. Annual Review Timeline

The academic year runs across four distinct review rounds per year, aligned perfectly with main and deferred/supplementary result releases.

February Round 1

Sem 2 Def / Sup Results

Stream 3 auto-downgrades to Stream 2. Requires Ad-Hoc process for hearings.
July Round 2

Sem 1 Main Results

Standard Stream 2 & Stream 3 (Hearings) processing rules apply.
August Round 3

Sem 1 Def / Sup Results

Stream 3 auto-downgrades to Stream 2. Requires Ad-Hoc process for hearings.
December Round 4

Sem 2 Main Results

Standard Stream 2 & Stream 3 (Hearings) processing rules apply.

2. Stream Execution Matrix

Both rulesets execute during all four rounds, but temporal conditional logic overrides system behavior in February and August.

Review Round Stream 2 Rules Match Stream 3 Rules Match Operational Outcome
February Stream 2 Notice Issued Downgraded to Stream 2 Ad-Hoc Manual Process required for Faculty Hearings.
July Stream 2 Notice Issued Stream 3 Hearing Triggered Standard automated system lifecycle applies.
August Stream 2 Notice Issued Downgraded to Stream 2 Ad-Hoc Manual Process required for Faculty Hearings.
December Stream 2 Notice Issued Stream 3 Hearing Triggered Standard automated system lifecycle applies.

3. Current Proposed Ad-Hoc Process

Preventing automation in February/August creates severe operational friction. If a faculty bypasses the downgrade, they must trigger this 7-step manual workflow:

1

Faculty Manual Identification

Faculty manually checks Stream 2 students before the CRM database main extract load to intercept and prevent the standard Stream 2 notice.

2

Manual Indicator Validation

Cross-checking standard profiles to confirm the student explicitly matches Stream 3 indicators to officially "qualify" them for referral.

3

Manual CRM Case Creation

Administrative teams must manually upload records into CRM to generate an active student case file.

4

Manual Email Communication

Dispatched manually outside core automated workflows. Includes a standalone support tool fallback link (Google Forms).

5

Monitor Google Form Submissions

Continuous manual observation of external form responses for submissions tracking.

6

Manual CRM Upload

Manually porting information collected from external Google Forms directly back into the CRM case profile.

7

Manual Faculty Hand-Off Notice

Issuing manual notifications to relevant Faculty points-of-contact stating that the student has completed pre-requisites and next procedural steps can begin.

Strategic Suggestion

Instead of enforcing system limitations that necessitate complex, human-managed exceptions, we should evaluate identifying Stream 3 across all 4 tracking cycles natively.

Instead of building programmatic code blocks to enforce downgrades, restrict when faculties are permitted to push students to hearings using institutional governance rules rather than physical code blocks.

System Design Advantage: Reduces technical complexity & architectural modifications.
Operations Advantage: Mitigates administrative overhead for central teams and faculties.

4. Policy Re-Draft Mapping

Transitioning legacy broad timelines to explicit boundary rules to better adjust for standard vs. deferred results workflows.

Legacy Progress Structure (2 Periods)

Period 2.2: Includes all teaching periods with results released after Semester 2 main release up to/including Semester 1 main release.

Period 2.3: Includes all teaching periods with results released after Semester 1 main release up to/including Semester 2 main release.

Proposed Operational Structure (4 Periods)

Academic Progress Period 1 (AP 1) From Semester 2 Main Release Semester 2 Deferred/Supplementary Release.
Academic Progress Period 2 (AP 2) From Semester 2 Deferred/Sup Release Semester 1 Main Release.
Academic Progress Period 3 (AP 3) From Semester 1 Main Release Semester 1 Deferred/Supplementary Release.
Academic Progress Period 4 (AP 4) From Semester 1 Deferred/Sup Release Semester 2 Main Release.
